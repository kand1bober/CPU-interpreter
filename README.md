# CPU-Interpreter

Учебный проект интерпретатора процессора на **C/C++** с простым **Ruby-ассемблером** для генерации бинарных программ и с поддержкой **JIT-перевода базовых блоков** через **AsmJit**.

Проект читает бинарный файл с инструкциями, декодирует команды, исполняет их в модели CPU и памяти, а для часто выполняемых базовых блоков умеет строить нативный x86-код.

## Возможности

- интерпретация пользовательского набора инструкций;
- загрузка бинарной программы из файла;
- выполнение операций с регистрами, памятью, ветвлениями и syscall;
- сборка тестовой программы на Ruby DSL;
- построение базовых блоков;
- JIT-перевод горячих блоков через AsmJit.

## Как это работает

### 1. Генерация бинарника программы

Файл `asm.rb` задаёт Ruby DSL для кодирования инструкций в бинарный формат.  
Скрипт `fib.rb` использует этот DSL и создаёт файл `executables/fib` с тестовой программой вычисления чисел Фибоначчи.

### 2. Запуск интерпретатора

Программа `cpu` получает имя бинарника как аргумент командной строки, загружает его из каталога `executables/`, инициализирует регистры и память и начинает цикл:

- `fetch`
- `decode`
- `execute`
- `advance_pc`

### 3. Базовые блоки и JIT

Во время исполнения проект собирает последовательности обычных инструкций в **base blocks**.  
Когда блок начинает часто выполняться, он переводится в нативный код через **AsmJit** и затем может исполняться уже как **translated block**.

## Сборка

```bash
cmake -B build
cmake --build build
```

Требования:

- CMake
- C++17 compiler
- Ruby
- AsmJit (подключён как submodule в `3rdparty/asmjit`)

## Пример запуска

### Сгенерировать тестовую программу

```bash
ruby fib.rb
```

### Собрать проект

```bash
cmake -B build
cmake --build build
```

### Запустить интерпретатор

```bash
./build/cpu fib
```

## Что находится в коде

- `main.cpp` — основной цикл исполнения, поиск/построение базовых блоков и запуск JIT;
- `interpret.cpp` — fetch/decode/execute, работа с памятью и `pc`;
- `translate.cpp` — перевод базовых блоков в машинный код через AsmJit;
- `instructions.hpp` — список инструкций и syscall;
- `asm.rb` — Ruby DSL для генерации бинарника;
- `fib.rb` — пример программы Фибоначчи.
